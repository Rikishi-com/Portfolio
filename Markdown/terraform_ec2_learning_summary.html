Terraform による EC2 構築と削除の学習まとめ

概要

本学習では，Terraform を用いて AWS 上に EC2 インスタンスを構築し，SSH 接続を行い，最後に terraform destroy によって削除する一連の操作を実施した．インフラのコード化（IaC）における基本的な構築・削除サイクルを体験した．

実施内容と使用ファイル

使用ファイル一覧

  • main.tf：リソース（EC2インスタンス）の定義
  • variables.tf：変数の定義
  • terraform.tfvars：変数に対する実際の値の指定
  • outputs.tf：EC2の出力情報（例：public_ip）を表示

各ファイルの内容

1. main.tf

基本となる設定ファイル．何をどこにどんな設定で作るかを設定するファイル

provider "aws" {
region = "ap-northeast-1"
}

resource "aws_instance" "web" {
    ami = var.ami_id
    instance_type = var.instance_type
    key_name = var.key_name

    tags = {
        Name = "TerraformExample"
    }
  
}

provider - 使用するクラウドプロバイダを指定（ここでは AWS）
region-リージョンを設定．今回は東京リージョン[ap-northeast-1]
resource - 実際に作成するリソースを定義
aws_instance - 作成するリソースの種類（AWS EC2 インスタンス）
web - リソースのローカル名。Terraform 内部で識別に使う
ami-任意のAMIを設定．
key_name - EC2 にSSH接続するためのキーペア名
tags - インスタンスにタグ（名前など）を付与するための設定．Name は表示されるインスタンス名

2.variables.tf

main.tfの変数を指定
.tfvarsとの違いは，variables.tfでは変数の定義をする．.tfvarsは具体的な値を記述する．
variables.tfはdefaultがあり，このおかげで.tfvarsが無くても，デフォルトの設定で動かすことが可能になる．

variable "instance_type" {
  default     = "t2.micro"
}

variable "ami_id"{
    description = "Amazon Linux 2023 AMI"
    default = "ami-0c1638aa346a43fe8"
}

variable "key_name" {
  description = "practice"
}

variable - 変数ブロックの開始を示す
default-.tfvarsで記述がなかった際にデフォルトで適用する設定
description-説明文．コメントとは異なり公式の情報を記載する

3.outputs.tf

リソースを作成後に必要な情報を出力するためのファイル

output "instance_public_ip" {
  description = "The public IP address of the EC2 instance"
  value       = aws_instance.web.public_ip
}

output - 出力したい値の定義ブロック
value - 出力する実際の値を指定
aws_instance.web.public_ip - 作成された EC2 インスタンスのパブリックIPアドレスを参照

補足　aws_instance.web.public_ip の構文について

これは Terraform における「リソースの属性参照」を意味する構文で，次の3つの要素で構成されている．

<リソースタイプ>.<ローカル名>.<属性>
部分 説明
aws_instance リソースの種類（AWS EC2 インスタンス）
web Terraform 内で定義されたローカル名（任意）
public_ip 作成された EC2 インスタンスのパブリックIPアドレスを指す属性

実行手順

1. 初期化 

terraform init

プロバイダプラグインなどをダウンロードし，Terraform プロジェクトを初期化する．

2. 実行プランの確認 

terraform plan

どのようなリソースが作成されるかを事前に確認する．

3. リソースの作成 

terraform apply

確認後，yes を入力して EC2 インスタンスを作成する．

4. IPアドレスの出力確認 

terraform output

EC2 インスタンスのパブリック IP アドレスなどが表示される．

5. SSH 接続 

chmod 400 practice.pem
ssh -i practice.pem ec2-user@<表示されたIP>

Amazon Linux である場合，ユーザー名は ec2-user である．

6. 削除 

terraform destroy

作成したリソースを削除する．

エラー対応

AMI ID に関するエラー 

InvalidAMIID.NotFound: The image id '[ami-XXXXXXXX]' does not exist

このエラーは，指定した AMI ID が現在のリージョンで無効であるか，存在しない場合に発生する．対応としては以下の通りである．

  • 正しいリージョンで使用可能な AMI ID を指定する
  • data "aws_ami" を使って動的に最新の AMI ID を取得する

学んだことまとめ

  • Terraform の構成ファイルは .tf で記述し，ファイル名に制約はない
  • main.tf にリソースを記述し，variables.tf に変数を定義できる
  • 値の指定は terraform.tfvars または -var オプションで行える
  • output を使って構築後の情報を出力できる
  • .pem ファイルは GitHub にアップロードしてはならない
  • terraform destroy でクリーンに削除できるため，検証に適している
