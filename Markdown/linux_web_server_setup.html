初心者向けLinuxサーバ立ち上げ時のコマンドまとめ

このページでは，初心者向けにLinuxサーバを作成する際に使用するコマンドをまとめました．
このページではAmazon Linuxサーバ　2023を使用しており，EC2サーバの立ち上げが完了している，もしくはそれに準ずる段階まで進んでいることを前提としています．

1. インストール関連

dnf -パッケージマネージャー 

dnf
  • 説明：dnfはAmazon LinuxなどのFedora，RedHat系で使われるパッケージマネージャーのこと
  • 注意；sudoを付けないと権限不足で失敗する
    sudo dnf

実際の手順

最新の状態にアップデートする

  • 管理者権限sudoを用いて，パッケージを最新にアップデートする
    sudo dnf update -y
    • -yはすべての確認メッセージに対してYesを選択するということ
    • ーyなしの場合は，アップデートされる事柄一つ一つに対して確認メッセージが出てくるので，自身で確認する
  • Complete! というメッセージが出てきたら成功です．

必要なパッケージをダウンロードする

  • 以下のコマンドを入力する
    ※暗記不要
    sudo dnf install -y httpd wget php-fpm php-mysqli php-json php php-devel
    • httpd：Apacheと呼ばれるWebサーバのこと．
    • wget：http/https経由でサーバからファイルを取得することが可能になるコマンドライン
    • php-fpm：PHP FastCGI Process Manager
      ApacheとPHPを連携させるための仕組み
    • php-mysqli：MySQL Improved
      PHPからMySQLを操作するための拡張機能
    • php：PHP本体
    • php-devel；PHPの開発ヘッダやビルドに必要なライブラリ

-v-バージョン確認方法

  • 上で説明したパッケージ名の後ろにつける

例

httpd -v

Webサーバ起動

  • 以下のコマンドでWebサーバ(今回はApache)を起動する

    sudo systemctl start httpd

    systemctl：システム管理ツール．以下にこのコマンドの重要な使い方をまとめる．

    systemctlの使い方

    基本記法

    sudo systemctl [コマンド名] [サービス名]
    コマンド 説明
    start サービス即時起動
    stop サービス停止
    restart サービス再起動
    status サービス状態確認
    enable サービス自動起動
    disable サービス自動起動無効
    is-active サービス起動中か確認
    is-enable サービス自動起動設定か確認

2．Apacheの設定

情報確認

次のコマンドを入力し，必要な情報を確認する

ls -l /var/www/
  • ls：ファイルやディレクトリを一覧表示する
  • -l：ロング形式のこと．詳細情報を表示する
  • /var/www/：対象ディレクトリ．多くのLinux環境ではWebサーバのドキュメントルートがここに存在する
  • ※ドキュメントルート ：WebページのHTML,CSS，画像などを配置する場所．ドキュメントルート以下のファイルだけが外部に公開される

アクセス制御（パーミッション）

上のコマンドを入力すると，メッセージが返ってくる．その際にそのフォルダのアクセス権限が確認できる．

表示されるアクセス権限の読み方

例えば以下のような表示があったとします：

drwxr-xr-x  2 root root 4096 Apr 10 12:00 html
部分 意味
d ディレクトリ
rwx 所有者（この場合root）の権限．read,write,x（実行）のこと
r-x グループ（この場合root）の権限，read,xのみ
2 リンク数（このディレクトリに何個のリンクがあるか）
root 所有者
root グループ
4096 ファイルサイズ
Apr 10 12:00 最終更新日時
html ディレクトリ名

3. Apacheの自動起動設定

起動時に自動的にApacheが立ち上がるようにする 

sudo systemctl enable httpd

自動起動設定の確認 

sudo systemctl is-enabled httpd
  • enabled と表示されればOK

4. ファイアウォールの設定（必要な場合）

Amazon Linux 2023では、firewalld はデフォルトで無効になっている場合もありますが、もし有効な場合はポートの開放が必要です。

HTTP（ポート80）の開放 

sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload

HTTPS（ポート443）の開放（SSLを使用する場合） 

sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

5. Webサーバの動作確認

ブラウザを開いて以下のようにアクセスします：

http://[自分のEC2インスタンスのパブリックIPアドレス]/

Apacheの初期ページ（例："Amazon Linux Apache test page"）が表示されれば成功です。

6. トラブルシューティング（基本）

Apacheが起動しない・動作しないとき

以下のコマンドでログを確認：

sudo journalctl -u httpd

状態を確認 

sudo systemctl status httpd

エラーが表示されている場合は、その内容をもとに対応する必要があります。

7. よく使う便利コマンド

コマンド 意味
cd ディレクトリ移動
ls -l ファイル・ディレクトリの詳細表示
pwd 現在のディレクトリを表示
mkdir ディレクトリ作成
rm ファイル削除
rm -r ディレクトリを中身ごと削除
cp ファイル・ディレクトリをコピー
mv ファイル・ディレクトリを移動または名前変更
cat ファイルの中身を表示
nano 簡易テキストエディタ（初心者向け）
vim 高機能なテキストエディタ（中級者以上向け）

補足：ドキュメントルートを変更したい場合

設定ファイル /etc/httpd/conf/httpd.conf を編集することでドキュメントルートの変更が可能です。

sudo nano /etc/httpd/conf/httpd.conf
  • DocumentRoot "/var/www/html" の箇所を変更し、同様に <Directory "/var/www/html"> も一致させて変更します。
  • 編集後はApacheを再起動：
sudo systemctl restart httpd

以上が基本的なLinux上でのApache Webサーバ構築手順です。
