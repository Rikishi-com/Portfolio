Terraform 初期設定手順（Mac向け）

AWSをTerraformで操作するための準備手順を、Macユーザー向けにまとめました。各ステップには用語解説や補足を加えています。

1. AWS CLI のインストールと確認

作業内容

AWS公式サイトからCLIをダウンロードしてインストール
以下のコマンドで正しくインストールされたか確認：

aws --version
  • AWS CLI-AWSサービスをコマンドラインで操作できるツール
  • Macでは公式のインストーラまたは brew install awscli で導入可能。

2. IAMユーザーの作成と設定

作業内容

目的 IAMユーザー名（例） 権限
AWS CLI操作用 cli-user AdministratorAccess（全権限）
Terraform操作用 terraform-user AdministratorAccess（全権限）

それぞれのユーザーにアクセスキー・シークレットキーを発行
以下のコマンドでプロファイル登録：

aws configure

複数プロファイルを使う場合：

aws configure --profile terraform
  • IAM（Identity and Access Management）：AWS上のアクセス権を管理するサービス。
  • プロファイル：AWS CLIで複数のアカウント・設定を切り替えるための名前付き設定。

3. Terraform のインストール（tfenv使用）

作業内容

Terraformをバージョン管理できる tfenv を使って導入：

git clone https://github.com/tfutils/tfenv.git ~/.tfenv
echo 'export PATH="$HOME/.tfenv/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

Terraformのインストールとバージョン確認：

tfenv install latest
tfenv use latest
terraform --version
  • tfenv-Terraformの複数バージョンを管理するためのツール。
  • latest と指定すれば最新版が入る。

4. GNU grep のインストール（互換性のため）

Macに最初から入っているgrepはBSD版である．Linuxと同様の操作で扱うため，GNU版をインストールする

brew install grep
echo 'export PATH="/opt/homebrew/opt/grep/libexec/gnubin:$PATH"' >> ~/.zshrc
source ~/.zshrc
  • brew-正式名称:Homebrew　Macで使えるパッケージマネージャー．AmazonLinux2023でいうdnfと同一の機能を持つ
  • grep-ファイルや標準出力から特定の文字列やパターンを検索するためのコマンド

5. git-secrets の導入（セキュリティ強化）

作業内容 

brew install git-secrets
git secrets --register-aws --global
git secrets --install ~/.git-templates/git-secrets
git config --global init.templatedir ~/.git-templates/git-secrets

補足

  • git-secrets-アクセスキーなどの漏洩を防ぐために，コミット前にスキャンするGit拡張機能
  • register-aws-AWS用のシークレット検出ルール
  • grobal-すべてのGitリポジトリで有効
  • install-テンプレート用のフックスクリプトを指定したパスにダウンロード
  • config-Gitのユーザー名・メールアドレス・エディタなどあらゆる動作設定を行うコマンド
